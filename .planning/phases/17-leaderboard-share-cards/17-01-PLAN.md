  <action>
Rewrite the GET handler to support three ranking views via `view` query param:

**Query params:** `view=founders|influence|trending`, `period=allTime|monthly|weekly` (ignored for trending), plus the existing auth Bearer token.

**Most Founders (`view=founders`):**
- Count rows in `founder_badges` table grouped by `fan_id`
- Join `fans` for name, avatar_url
- Apply date filter on `created_at` if period != allTime
- Sort by count DESC, limit 50
- Return rank, fanId, name, avatarUrl, metric (founder count)

**Highest Influence (`view=influence`):**
- For each fan, find all performers they founded (from `founder_badges`)
- Count how many OTHER users collected those performers (from `collections` where `fan_id != founder_fan_id`)
- That sum = influence score
- Apply date filter on collections.created_at if period != allTime
- Sort by influence DESC, limit 50
- Return rank, fanId, name, avatarUrl, metric (influence score)

**Trending (`view=trending`):**
- Count `founder_badges` created in the current calendar week (Monday to now) grouped by fan_id
- Always current week, ignore period param
- Sort by count DESC, limit 50
- Return rank, fanId, name, avatarUrl, metric (weekly founders count)

**User position:** For all views, also compute the requesting user's own rank and metric. If the user is NOT in the top 50, include their entry as `userPosition: { rank, fanId, name, avatarUrl, metric }`. If they ARE in the top 50, set `userPosition: null`.

**Response shape:**
```json
{
  "entries": [{ "rank": 1, "fanId": "...", "name": "...", "avatarUrl": "...", "metric": 42 }],
  "userPosition": { "rank": 55, "fanId": "...", "name": "...", "avatarUrl": "...", "metric": 3 } | null
}
```

Look up the requesting user's fan_id from their email (same pattern as existing: `getAuthEmail` -> lookup in fans table by email).

Keep the existing auth pattern. Remove the old fans/performers tab logic entirely. Default `view` to `founders` if not specified.
