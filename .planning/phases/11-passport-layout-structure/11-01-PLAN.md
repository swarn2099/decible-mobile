**Sticky tab bar behavior:**

The PassportPager receives `scrollY` and `headerHeight` from the parent. Instead of the pager owning the scroll, the parent owns it. The approach:

In passport.tsx, use a single `Animated.ScrollView` wrapping: [header, tab bar, pager content]. Use `stickyHeaderIndices={[1]}` to make the tab bar (index 1) stick when scrolled. BUT — PagerView inside ScrollView is problematic on Android.

**Better approach — collapsible header pattern:**
- passport.tsx renders: header (absolute or in flow) + PassportPager (flex: 1).
- PassportPager's internal ScrollViews each get `onScroll` handler that updates `scrollY`.
- Header uses `useAnimatedStyle` to translate up by `scrollY` (clamped to headerHeight).
- Tab bar sits between header and pager. It uses `useAnimatedStyle`: when `scrollY >= headerHeight`, it stays pinned at top:0 (using `translateY: Math.max(0, scrollY - headerHeight)` to counteract the scroll, but actually simpler with position logic).

**Simplest correct approach:**
1. passport.tsx uses a `Animated.FlatList` or custom approach: The header and tab bar are rendered INSIDE each PagerView page's ScrollView via `ListHeaderComponent` or explicit rendering. This way scroll naturally pushes header up and tab bar sticks.

Actually, the cleanest pattern for this exact use case (header + sticky tabs + PagerView):

**Use `react-native-collapsible-tab-view` pattern manually:**

In passport.tsx:
- Render header as a fixed-height View.
- Below it, render the tab bar.
- Below that, render PagerView with flex: 1.
- Track `scrollY` from each page's inner ScrollView using `useAnimatedScrollHandler`.
- Apply `translateY: -Math.min(scrollY.value, HEADER_HEIGHT)` to a wrapper containing [header + tab bar + pager] — NO, this shifts everything.

**Final approach (proven pattern):**

The entire passport screen is structured as:
```
<View flex:1 bg:colors.bg>
  <SafeAreaView edges={["top"]}>
    <Animated.View style={headerAnimatedStyle}>
      <PassportHeader ... />        {/* ~180px, this slides up */}
    </Animated.View>
    <View>                           {/* Tab bar — stays visible */}
      <TabBar ... />
    </View>
    <PagerView flex:1>
      {pages with internal ScrollViews}
    </PagerView>
  </SafeAreaView>
</View>
```

Each page's ScrollView uses `useAnimatedScrollHandler` to update `scrollY`. The header wrapper has:
```typescript
const headerStyle = useAnimatedStyle(() => ({
  height: interpolate(scrollY.value, [0, HEADER_HEIGHT], [HEADER_HEIGHT, 0], Extrapolation.CLAMP),
  overflow: 'hidden',
  opacity: interpolate(scrollY.value, [0, HEADER_HEIGHT * 0.7], [1, 0], Extrapolation.CLAMP),
}));
```

This collapses the header as content scrolls, and the tab bar naturally pins to the top when header is fully collapsed. The PagerView pages' ScrollViews need `contentContainerStyle={{ paddingTop: 0 }}` and each uses `scrollEventThrottle={16}`.

Synchronize scroll positions across tabs: when switching tabs, the new tab's scroll position should match the header collapse state. Use a `scrollOffsets` ref array (one per tab) to track each tab's position.

When `scrollY >= HEADER_HEIGHT`, add a subtle 1px bottom border on the tab bar (colors.divider) to indicate pinned state — use `useAnimatedStyle` with `borderBottomWidth` or opacity on a separator view.
