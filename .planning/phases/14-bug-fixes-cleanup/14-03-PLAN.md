<task type="auto">
  <name>Task 1: Fix stat count mismatch between search results and artist profiles</name>
  <files>src/hooks/useSearch.ts, src/components/search/SearchResultCard.tsx</files>
  <action>
  **Root cause analysis:**
  The `SearchResultCard` displays `performer.follower_count` which comes from the `performers` table column. This column stores the PLATFORM follower count (e.g., Spotify monthly listeners or SoundCloud followers) — NOT the Decibel fan count. Meanwhile, the artist profile page uses `useArtistFanCount` which calls `/mobile/artist-stats` API and returns the actual count of collections (Decibel fans). These are two completely different numbers.

  The CONTEXT.md says "Brendan shows 58 finds when searched but 17 on his profile" — this likely means the search shows `follower_count` (platform number) while the profile shows the Decibel fan count from collections.

  **Fix approach — make search cards show the Decibel fan count:**

  1. In `useSearch.ts`, update the `useDecibelSearch` query to include a count of collections for each performer. Change the Supabase query to join with the `collections` table:
     ```typescript
     const { data, error } = await supabase
       .from("performers")
       .select("id, name, slug, photo_url, genres, collections(count)")
       .ilike("name", `%${query}%`)
       .limit(10);
     ```
     Then map the result to extract `fan_count` from `collections[0].count`.

  2. Update `DecibelSearchResult` type: rename `follower_count` to `fan_count` (or add `fan_count` alongside it).

  3. In `SearchResultCard.tsx`, display the `fan_count` (Decibel fans) instead of `follower_count`. The icon (Users) and label format stays the same.

  4. If Supabase `.select("collections(count)")` doesn't work cleanly with the count aggregation, an alternative approach: use a Supabase RPC or just do a separate count query. Simplest fallback: keep `follower_count` from performers table but label it differently (e.g., show it as "listeners" instead of fans). However, the correct fix is to show the Decibel fan count.

  5. Verify the fix works with at least 3 different artists by comparing the search card number with the artist profile fan count.
