  **Backend validate-artist-link/route.ts:**
  1. Update `parseUrl` function to handle song/album URLs:
     - Spotify track: match `open.spotify.com/track/([a-zA-Z0-9]+)` → set `contentType: "track"`, store trackId
     - Spotify album: match `open.spotify.com/album/([a-zA-Z0-9]+)` → set `contentType: "album"`, store albumId
     - Apple Music album: match `music.apple.com/[region]/album/[name]/[id]` without `/artist/` → parse as album
     - Apple Music song: album URL with `?i=trackId` query param → parse as song
     - SoundCloud track: match `soundcloud.com/[username]/[track-slug]` (2 segments) → parse as track
  2. Update `ParsedUrl` interface: add `contentType?: "artist" | "track" | "album"` and `contentId?: string`.

  4. For Spotify albums: use Spotify oEmbed or Web API `GET /albums/{id}` which returns `artists[0]`. Then resolve artist ID.
  5. For Apple Music songs/albums: use iTunes Lookup API `https://itunes.apple.com/lookup?id={albumId}&entity=song` to get the artist name from the results. Then cross-reference on Spotify by name (existing `probeSpotifyByName` function).
  6. For SoundCloud tracks: use the existing SoundCloud resolve API with the full track URL — it returns `user.username` and `user.avatar_url` which gives us the artist. Extract artist username from the resolved data.
  7. Add `found_via_track?: string` to the `ValidateResponse` type. When the input was a song URL, include the track name in the response so the mobile app can show "Found via [Track Name]".
  8. After resolving the artist from a track/album, continue with the existing eligibility check, existing performer lookup, and relationship check — the rest of the flow is identical.
