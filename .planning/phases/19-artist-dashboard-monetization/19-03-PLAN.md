1. GET /api/dashboard/overview:
   - Auth required: get user, look up their artist_claims record to get performer_id
   - Return 401 if no claim, 403 if not verified
   - Query collections table for this performer_id:
     a. Total collector count: SELECT COUNT(DISTINCT fan_id) FROM collections WHERE performer_id = X
     b. Collector growth (last 90 days): Group by date, return { date, count } array for cumulative growth chart
        - Get all collection dates, bucket into days, compute running total
     c. Recent activity (last 20): Join collections with fans table, return fan_name, fan_avatar_url, collection_type, created_at
   - Also fetch artist_subscriptions for this performer to include subscription status
   - Return DashboardOverview shape

2. GET /api/dashboard/fan-intelligence:
   - Auth required same as above
   - Query:
     a. Full collector list: Join collections + fans + founder_badges, for each fan include: fan_id, name, avatar_url, city, collected_at, is_founder (exists in founder_badges), also_collects (up to 3 other performer names this fan has collected)
        - For "also_collects": subquery collections WHERE fan_id = X AND performer_id != current_performer, join performers for names, LIMIT 3
     b. City breakdown: GROUP BY fans.city, count, order by count DESC, limit 10
     c. Fans also collect: For all fans who collected this artist, find other artists they also collected, count overlaps, return top 10 by overlap_count with artist name and slug
   - Return FanIntelligence shape
   - NOTE: This endpoint can be slow (multiple joins). Use Promise.all for parallel queries. Limit collector list to 200.
  </action>
