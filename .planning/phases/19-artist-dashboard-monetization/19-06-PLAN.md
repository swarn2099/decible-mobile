Create a single route file with GET, POST, PUT, DELETE handlers:

1. GET /api/dashboard/links:
   - Auth required: user -> artist_claims -> performer_id
   - Query artist_links WHERE artist_id = performer_id ORDER BY display_order ASC
   - Also return the "auto-detected" links from performers table (spotify_url, etc.) so the UI can show what's auto-detected vs manually added
   - Return { custom_links: ArtistLink[], auto_links: { platform, url }[] }

2. POST /api/dashboard/links:
   - Auth required
   - Body: { platform: string, url: string }
   - Validate URL format (must start with http:// or https://)
   - Validate platform is a known value: 'spotify', 'apple_music', 'soundcloud', 'mixcloud', 'instagram', 'twitter', 'youtube', 'tiktok', 'website', 'bandcamp', 'other'
   - Get max display_order for this artist, insert with display_order = max + 1
   - Return created ArtistLink

3. PUT /api/dashboard/links:
   - Auth required
   - Body: { links: { id: string, display_order: number }[] } (for reordering)
   - Validate all link IDs belong to this artist
   - Batch update display_order for each link
   - Return { success: true }

4. DELETE /api/dashboard/links:
   - Auth required
   - Body: { link_id: string }
   - Validate link belongs to this artist
   - Delete from artist_links
   - Return { success: true }
