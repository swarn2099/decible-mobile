**Step 2 — Create `~/decibel/src/app/api/mobile/validate-artist-link/route.ts`:**

This is a new POST endpoint. Follow the exact auth pattern from `add-artist/route.ts` (Bearer token -> getAuthEmail -> fan lookup).

**Request body:** `{ url: string }`

**Logic flow:**
1. Authenticate via Bearer token, get fan_id
2. Parse the URL to detect platform:
   - `open.spotify.com/artist/{id}` or `spotify:artist:{id}` -> Spotify
   - `spotify.link/{code}` -> follow redirect server-side with `fetch(url, { redirect: "manual" })`, extract Location header, parse as Spotify URL
   - `soundcloud.com/{username}` (strip www/m subdomains) -> SoundCloud
   - `music.apple.com/.../artist/{name}/{id}` or `itunes.apple.com/.../artist/{name}/{id}` -> Apple Music
   - Anything else -> return `{ eligible: false, rejection_reason: "unsupported_platform" }`
3. Resolve artist data per platform:
   - **Spotify:** Call `getSpotifyArtist(artistId)` + `scrapeMonthlyListeners(artistId)`. Gate: if `monthly_listeners !== null && monthly_listeners >= 1_000_000` -> reject. If `monthly_listeners === null` -> pass through as eligible (unverified).
   - **SoundCloud:** Call SoundCloud widget API (same pattern as resolve-link, using `SOUNDCLOUD_CLIENT_ID`). Gate: if `followers_count >= 100_000` -> reject.
   - **Apple Music:** Extract artist name from URL path. Call `probeSpotify(name)` to cross-reference. If Spotify match found and monthly listeners >= 1M -> reject. If no Spotify match -> default eligible per PRD.
4. Check if artist already exists in Decibel: query `performers` table by spotify_id, soundcloud URL, or name match. Use the `findExistingPerformer` pattern from resolve-link (extract/adapt it, don't import from the route file).
5. If existing performer found, check user relationship:
   ```typescript
   const [collectionsRow, founderRow] = await Promise.all([
     admin.from("collections").select("id").eq("fan_id", fanId).eq("performer_id", performerId).maybeSingle(),
     admin.from("founder_badges").select("fan_id").eq("performer_id", performerId).maybeSingle(),
   ]);
   const isFounder = founderRow?.fan_id === fanId;
   const hasCollected = !!collectionsRow;
   // Determine: founded > collected > discovered > none
   ```
6. Return the response shape documented in the interfaces block above.

**Important:**
- Use `createClient` from `@supabase/supabase-js` with service role key (same pattern as other mobile routes)
- Import `getSpotifyArtist` and `scrapeMonthlyListeners` from `~/decibel/src/lib/spotify.ts`
- Extract/adapt `findExistingPerformer` and `probeSpotify` logic from resolve-link (do NOT import the route file)
- Handle network errors gracefully: if Spotify/SoundCloud API fails, return a 502 with `{ error: "Failed to fetch artist data" }`
- The `spotify.link` redirect resolution: try `fetch(url, { redirect: "manual" })` and read the `Location` header. If that fails, try `fetch(url, { redirect: "follow" })` and parse `response.url`.
