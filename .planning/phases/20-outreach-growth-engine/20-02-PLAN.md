**4. src/outreach-cron.ts:** Daily outreach job using node-cron.

Schedule: `0 10 * * *` (10 AM UTC daily — 5 AM CT, emails arrive in the morning)

Logic:
1. Query performers with 10+ collectors who have NO row in artist_outreach with outreach_type='initial':
```sql
SELECT p.id, p.name, p.slug, p.image_url, p.email, p.instagram_handle,
       COUNT(c.id) as collector_count
FROM performers p
JOIN collections c ON c.performer_id = p.id
LEFT JOIN artist_outreach ao ON ao.artist_id = p.id AND ao.outreach_type = 'initial'
WHERE ao.id IS NULL
GROUP BY p.id
HAVING COUNT(c.id) >= 10
ORDER BY COUNT(c.id) DESC
```

Use Supabase client RPC or raw query. If RPC is complex, use the Supabase JS client with joins:
- Query collections grouped by performer_id with count >= 10
- Then filter out performers already in artist_outreach

2. For each qualifying artist, get the founder (first person to add them):
```sql
SELECT f.name, f.slug FROM founder_badges fb
JOIN fans f ON f.id = fb.fan_id
WHERE fb.performer_id = $artist_id
LIMIT 1
```

3. Determine channel:
- If `p.email` exists: channel = 'email', send via sendOutreachEmail, status = 'sent'
- If `p.instagram_handle` exists (no email): channel = 'instagram_dm', status = 'pending'
- If neither: channel = 'manual', status = 'pending'

4. Generate the message_text using the template (always, regardless of channel — for IG DM copy-paste).

5. Insert into artist_outreach (the UNIQUE index prevents duplicates even if the cron runs twice):
```
{ artist_id, channel, outreach_type: 'initial', message_text, collector_count_at_send, status, sent_at: (if email sent, NOW()) }
```

6. Log: "[DATE] Outreach: X artists identified, Y emails sent, Z IG DMs queued, W manual flagged"

Add a `--dry-run` CLI flag that logs what would happen without inserting/sending.

**1. src/milestone-cron.ts:** Milestone notification job using node-cron.

Schedule: `0 11 * * *` (11 AM UTC daily — runs 1 hour after outreach cron)

Logic:
1. Query all performers with their collector counts:
```sql
SELECT p.id, p.name, p.slug, p.image_url, p.email,
       COUNT(c.id) as collector_count
FROM performers p
JOIN collections c ON c.performer_id = p.id
GROUP BY p.id
HAVING COUNT(c.id) >= 25
```

2. For each artist, check which milestone thresholds they've crossed (25, 50, 100):
- If collector_count >= 25 and no artist_outreach row with milestone_threshold=25: create milestone
- If collector_count >= 50 and no artist_outreach row with milestone_threshold=50: create milestone
- If collector_count >= 100 and no artist_outreach row with milestone_threshold=100: create milestone

3. For each new milestone:
- Generate share card URL: `${DECIBEL_BASE_URL}/api/share-card/milestone?artistName=${encodeURIComponent(name)}&collectorCount=${count}&milestone=${threshold}&artistSlug=${slug}&artistPhoto=${encodeURIComponent(image_url)}`
- If email exists: send milestone email with share card URL, set status='sent'
- If no email: insert with status='pending', channel='manual'
- message_text: milestone-specific congratulations + share card link + claim CTA

4. Insert into artist_outreach:
```
{ artist_id, channel, outreach_type: 'milestone_${threshold}', message_text, collector_count_at_send, milestone_threshold: threshold, status, sent_at }
```

The UNIQUE index on (artist_id, milestone_threshold) WHERE milestone_threshold IS NOT NULL prevents duplicates.

5. Log: "[DATE] Milestones: X new milestones detected (Y at 25, Z at 50, W at 100), A emails sent"

Support `--dry-run` and `--run-now` flags same as outreach cron.
