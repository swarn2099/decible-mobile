SQL for artist_outreach table:
```sql
CREATE TABLE IF NOT EXISTS artist_outreach (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  artist_id UUID NOT NULL REFERENCES performers(id),
  channel TEXT NOT NULL, -- 'email', 'instagram_dm', 'manual'
  outreach_type TEXT NOT NULL DEFAULT 'initial', -- 'initial', 'milestone_25', 'milestone_50', 'milestone_100'
  message_text TEXT,
  collector_count_at_send INTEGER NOT NULL,
  milestone_threshold INTEGER, -- NULL for initial, 25/50/100 for milestones
  status TEXT NOT NULL DEFAULT 'pending', -- 'pending', 'sent', 'claimed', 'ignored'
  sent_at TIMESTAMPTZ,
  claimed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Prevent duplicate outreach: one initial contact per artist, one per milestone threshold
CREATE UNIQUE INDEX IF NOT EXISTS idx_artist_outreach_initial
  ON artist_outreach (artist_id) WHERE outreach_type = 'initial';
CREATE UNIQUE INDEX IF NOT EXISTS idx_artist_outreach_milestone
  ON artist_outreach (artist_id, milestone_threshold) WHERE milestone_threshold IS NOT NULL;

-- Index for cron job queries
CREATE INDEX IF NOT EXISTS idx_artist_outreach_status ON artist_outreach (status);
CREATE INDEX IF NOT EXISTS idx_artist_outreach_artist ON artist_outreach (artist_id);
```

Also create the SQL migration file at the supabase/migrations path for version control.

After creating the endpoint, deploy with `cd /home/swarn/decibel && vercel --prod` (or git push), then hit the endpoint to run the migration: `curl https://decible.live/api/admin/migrate-phase20`.
