async function getAuthEmail(req: NextRequest): Promise<string | null> {
  const auth = req.headers.get("authorization");
  if (!auth?.startsWith("Bearer ")) return null;
  const { data, error } = await admin.auth.getUser(auth.slice(7));
  if (error || !data.user?.email) return null;
  return data.user.email;
}

  <action>
Create two new API endpoints in the decibel backend repo (NOT decibel-mobile).

**GET /api/mobile/user-stats** — Returns stats for the authenticated user's stats bar:
1. Auth: use the same `getAuthEmail` + admin createClient pattern from existing routes
2. Look up fan by email
3. Query:
   - `finds_count`: COUNT of collections WHERE fan_id = fan.id (all collection types — founded + discovered + collected)
   - `founders_count`: COUNT of founder_badges WHERE fan_id = fan.id
   - `influence_score`: COUNT of collections by OTHER fans WHERE performer_id IN (SELECT performer_id FROM founder_badges WHERE fan_id = fan.id). This counts how many times other people have collected artists that this user founded. If user has no founder badges, influence = 0.
4. Return: `{ finds: number, founders: number, influence: number }`

**GET /api/mobile/trending-artists** — Returns most collected artists this week:
1. Auth: same pattern
2. Query collections from the last 7 days, group by performer_id, count collections per performer, ORDER BY count DESC, LIMIT 10
3. Join with performers table to get name, slug, photo_url
4. Return: `{ artists: Array<{ id, name, slug, photo_url, collector_count }> }`

Use raw Supabase queries. For influence_score, use two separate queries (get founder performer_ids, then count collections on those performers excluding the user). Do NOT use RPC or stored procedures — keep it in JS.
