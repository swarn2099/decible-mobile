<task type="auto">
  <name>Task 2: QA pass — hardcoded colors + bottom padding</name>
  <files>Multiple files across app/ and src/components/</files>
  <action>
Run a systematic grep-based QA audit and fix all findings.

**Step 1: Hardcoded hex colors (POL-01)**

```bash
grep -rn "#[0-9a-fA-F]\{6\}\|#[0-9a-fA-F]\{3\}" app/ src/components/ --include="*.tsx" | grep -v "//.*#" | grep -v "colors\." | grep -v "node_modules" | grep -v "__tests__"
```

For each finding:
- If it's a theme-dependent color (text, background, border, shadow), replace with `colors.X` from useThemeColors()
- If it's an accent color (pink #FF4D6A, purple #9B6DFF, gold #FFD700), these are OK as constants but should ideally reference `colors.pink`, `colors.purple`, `colors.gold`
- If the file doesn't import useThemeColors, add the import and hook call
- EXCEPTION: files that are purely about defining colors (like theme.ts, colors.ts) are OK to skip

**Step 2: Missing useThemeColors (POL-01)**

```bash
grep -rL "useThemeColors" app/ src/components/ --include="*.tsx" 2>/dev/null
```

For each screen/component file that renders UI but doesn't use useThemeColors:
- Add `import { useThemeColors } from "@/src/theme"` (or wherever the hook is)
- Replace any hardcoded color values with theme colors

**Step 3: Bottom padding for floating tab bar (POL-02)**

```bash
grep -rn "contentContainerStyle" app/ src/components/ --include="*.tsx"
```

For each ScrollView/FlatList/SectionList:
- Ensure `contentContainerStyle` includes `paddingBottom: 100` (minimum)
- If `contentContainerStyle` exists but `paddingBottom` is less than 100, increase it
- If `contentContainerStyle` is missing, add it with `{ paddingBottom: 100 }`
- Tab screens (app/(tabs)/*.tsx) need this the most since they sit above the floating pill

Also check for screens that use a View with `flex: 1` but have scrollable content at the bottom — these need bottom margin or padding.

**Step 4: Localhost references**

```bash
grep -rn "localhost\|127\.0\.0\.1" app/ src/ --include="*.tsx" --include="*.ts" --include="*.env*"
```

Remove any localhost references in production code paths.

**Step 5: Verify**
After all fixes, run `npx expo export --platform ios` to confirm build still passes.

Document all changes made in the summary.
