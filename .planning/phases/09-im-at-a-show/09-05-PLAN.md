**ConfidenceLineupScreen.tsx** — Three-tier confidence UI:
- Props: `{ result: ShowSearchResult; onCollect: (performerIds: string[]) => void; onManualFallback: () => void; onBack: () => void }`
- **High confidence** (`result.confidence === 'high'`):
  - Venue name displayed prominently at top
  - Artist list with checkboxes (all checked by default)
  - Each artist shows name + photo (if performer_id exists, fetch from DB) + Founder star if `is_founder_available`
  - "Collect All" pink button at bottom (calls onCollect with all performer_ids)
  - Essentially mirrors existing LineupStep but for scrape results
- **Medium confidence** (`result.confidence === 'medium'`):
  - "Does this look right?" header
  - Same artist list with checkboxes (all checked)
  - "Yes, Collect" button (primary pink) + "No, enter manually" text button (calls onManualFallback)
  - Source attribution: "Found via {result.source}" in muted text
- **Low confidence** (`result.confidence === 'low'`):
  - "We found some possibilities" header
  - Artist names listed but NOT checkable — each has a "Paste link to verify" prompt
  - For each artist: TextInput for platform link paste, uses `useValidateArtistLink` to validate
  - Only validated artists can be collected
  - "Enter manually instead" text button (calls onManualFallback)
- Theme: useThemeColors(), Poppins font, consistent padding (16px horizontal)
- Haptic: light impact on checkbox toggle

**ManualFallbackForm.tsx** — Venue autocomplete + artist link paste:
- Props: `{ lat: number; lng: number; localDate: string; onStamped: (stamps: StampData[]) => void; onBack: () => void }`
- Layout:
  - "Add manually" header
  - **Venue field**: TextInput with autocomplete. As user types, query `venues` table via Supabase for fuzzy match on name. Show dropdown of matches. If no match, user can type a new venue name.
  - **Artist link paste**: TextInput for Spotify/Apple Music/SoundCloud URL. Uses `useValidateArtistLink` to validate. Shows artist preview card after validation (name + photo + eligibility).
  - **"Check In" button**: disabled until venue (selected or new) AND at least one validated artist
  - On submit:
    1. If new venue: create via tag-performer endpoint (which handles new venue creation) OR direct Supabase insert
    2. Call PUT /api/mobile/show-checkin with venue_id + performer_ids + local_date
    3. Also insert into `venue_submissions` table for crowdsource tracking: `{ fan_id, venue_name, venue_id, lat, lng, performer_name, platform_url, event_date }`
    4. On success: call onStamped with stamp results
- Theme: useThemeColors(), both dark and light mode

- Props: `{ stamps: StampData[]; founders: string[]; venueName: string; eventDate: string; onViewPassport: () => void; onDone: () => void }`

    - Otherwise: pink check + "Stamped" label
  - Bottom buttons: "View Passport" (primary pink) + "Done" (text button)
- Theme: useThemeColors(), Poppins, consistent spacing
