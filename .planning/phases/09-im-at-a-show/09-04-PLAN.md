State machine flow:
1. `startCheckin(lat, lng)`: POST to `/mobile/show-checkin` with `{ lat, lng, local_date: getLocalDate() }`
2. If response `status === 'found'`: transition to `layer1_hit` with venue + enriched performers
3. If response `status === 'searching'`: transition to `waiting` with searchId
4. In `waiting` phase:
   - Start elapsed timer (useRef + setInterval every 1000ms, update state.elapsed)
   - Subscribe to Supabase Realtime:
     ```typescript
     const channel = supabase
       .channel(`search:${searchId}`)
       .on('postgres_changes', {
         event: 'INSERT',
         schema: 'public',
         table: 'search_results',
         filter: `search_id=eq.${searchId}`,
       }, (payload) => {
         setState({ phase: 'result', result: payload.new });
         cleanup();
       })
       .subscribe((status) => {
         if (status === 'CLOSED' || status === 'TIMED_OUT') {
           startPolling();
         }
       });
     ```
   - **Polling fallback** (for iOS background disconnect):
     ```typescript
     function startPolling() {
       pollRef.current = setInterval(async () => {
         const { data } = await supabase
           .from('search_results')
           .select('*')
           .eq('search_id', searchId)
           .maybeSingle();
         if (data) {
           setState({ phase: 'result', result: data });
           cleanup();
         }
       }, 3000);
     }
     ```
   - **Timeout**: At 15 seconds elapsed, if still in `waiting`, transition to `timeout`
   - **Cleanup function**: unsubscribe channel, clear poll interval, clear elapsed timer
5. Return `{ state, startCheckin, reset }` — reset clears all state back to idle

CRITICAL: All intervals and subscriptions MUST be cleaned up in the useEffect return function AND in the cleanup helper. Use refs for interval IDs to avoid stale closures.
